# notes.nvim

*[English version](README.md)*

Лёгкий плагин для Neovim, который ведёт заметки в отдельной вкладке — по образу
приложения **macOS Notes** — с опциональной синхронизацией с GitHub через Git (SSH).

<video src="https://github.com/user-attachments/assets/7ff882ff-7cfc-4739-8765-e9e4ddc6fd5f" autoplay loop muted playsinline></video>

```
────────────────────────────────────────────────
 Explorer                          ← статуслайн
 ▼ Notes/
   ▼ Work/
     ▶ Projects/
     10.07.2026 - Отчёт
   ▶ Personal/
   26.06.2026 - Список покупок
────────────────────────────────────────────────
 Notes/Список покупок               ← статуслайн (папка/заголовок)
 # Список покупок

 - [ ] Молоко
 - [ ] Позвонить в банк
```

Папки вкладываются на любую глубину и раскрываются/сворачиваются на месте
(`o` или `<CR>`), включая саму корневую строку `Notes/`:

```
────────────────────────────────────────────────
 Explorer
 ▼ Notes/
   ▼ Work/
     ▼ Projects/
       ▶ Q3/
       26.06.2026 - Заметки спринта
     ▶ Archive/
────────────────────────────────────────────────
```

## Возможности

- **Единый проводник-дерево + редактор** — открывается в новой полноэкранной вкладке: одно дерево-**проводник** (папки и заметки вперемешку, любая глубина) сверху, редактор снизу. Закрытие заметок закрывает вкладку.
- **Заголовок из содержимого** — у заметки нет задаваемого вручную имени файла. Её заголовок — это **первая непустая строка** текста; пустая заметка называется **«New Note»** и всегда закреплена вверху своей папки. На диске каждая заметка — это непрозрачный ID-файл (расширение `.md`, например `20260627143000.md`), поэтому изменение заголовка не засоряет историю git и не вызывает коллизий. Строки заметок показывают `дд.мм.гггг - Заголовок`, отсортировано по времени изменения (сначала новые) внутри каждой папки. **Заголовок обновляется на лету по мере набора текста**, без сохранения.
- **Корневая строка, вложенные папки, раскрытие на месте** — дерево всегда начинается со строки `Notes/` (корень репозитория); реальные папки вкладываются под неё на любую глубину и по умолчанию свёрнуты. Нажмите `o` (или `<CR>`) на любой папке — включая саму `Notes/` — чтобы раскрыть/свернуть её прямо в дереве, с отступом в 2 пробела на уровень; на заметке — чтобы перейти в фокус редактора. Соседние элементы сортируются так, что папка/заметка, в чьём поддереве самая недавно изменённая заметка, идёт первой, причём папки на одном уровне всегда идут раньше заметок. Строки папок показывают глиф закрыто/открыто (настраивается, см. `config.tree_icons`); у заметок по умолчанию иконки нет. Новые заметки (`a`) и папки (`A`) создаются внутри папки под курсором (или папки заметки под курсором — «контекстной папки»); вставка на саму строку `Notes/` нацеливает действие явно в корень. Пустые папки поддерживаются через скрытый `.gitkeep`, чтобы они коммитились и синхронизировались.
- **Перемещение курсором** — нажмите `x` на заметке или папке, чтобы пометить её (подсвечивается цветом выделения); повторное нажатие `x` на том же элементе отменяет перемещение. Перемещайтесь по дереву куда угодно (раскрывая/сворачивая по пути) к папке-приёмнику и нажмите `p`, чтобы вставить туда помеченный элемент (или на строку `Notes/`, чтобы вставить в корень). Папка-приёмник автоматически раскрывается, чтобы перемещённый элемент был виден, и курсор встаёт на него. Папку нельзя переместить в саму себя, в одну из своих подпапок, а корневую строку `Notes/` нельзя переместить вовсе.
- **Полноширинное выделение строки, без отвлекающего курсора** — окно проводника подсвечивает всю ширину строки под курсором (нативный `cursorline`); сам текстовый курсор терминала скрывается, пока фокус в проводнике, чтобы не перекрывать иконки дерева. Текущая *открытая* заметка при этом сохраняет собственную полноширинную подсветку независимо от положения курсора.
- **Нативное редактирование** — окно редактора ведёт себя как обычное окно `markdown`-файла (`number`, `cursorline`, `signcolumn`), поэтому глобальные стили на `InsertEnter`/`InsertLeave` и плагины статуслайна работают внутри него.
- **Мгновенное обновление интерфейса** — дерево обновляется сразу при `:w` (порядок сортировки, заголовок); синхронизация git идёт в фоне.
- **Полное управление** — `a` создаёт заметку, `A` создаёт папку (обе — внутри контекстной папки), `d` удаляет элемент под курсором, `x` помечает его для перемещения, `p` вставляет; папки также поддерживают переименование (`r`); обновление (`R`). **Каждое создание/удаление/перемещение/переименование сразу коммитится и пушится в GitHub.**
- **Настраиваемые сочетания клавиш** — каждое действие, клавиша закрытия и клавиша переключения панели переопределяются через `config.keys`.
- **Синхронизация Git** — при первом открытии: `git clone` (если каталога нет), затем `git pull` (merge). При `:w`: commit + merge + push. При любом создании/удалении/перемещении/переименовании: немедленный commit + merge + push. При закрытии (`q`): commit + merge + push оставшихся изменений.
- **Конфликты остаются в файле** — конфликт слияния оставляется обычными git-маркерами прямо в заметке (без диалога). Заголовок конфликтной заметки и имя её папки подчёркиваются волнистой линией цвета ошибки (подсветка папки покрывает конфликт в любом месте её поддерева); уберите маркеры, сохраните — слияние завершится и запушится. Перемещение/переименование/удаление конфликтной заметки блокируется до разрешения.
- **Запрос о несохранённых изменениях** — нажатие `q` при несохранённых изменениях в редакторе показывает диалог **Save / Discard / Cancel** вместо тихого сохранения или сброса. Выбор **Discard** перезагружает сохранённую версию с диска.
- **Защита от сбоев** — при каждом открытии отслеживаемые файлы, удалённые вне плагина (например, случайным `rm`), восстанавливаются из последнего коммита перед любым push, так что пустое рабочее дерево никогда не попадёт в удалённый репозиторий.
- **Иконка статуса синхронизации** — в заголовке вкладки Neovim рядом с `notes.nvim` отображается индикатор: анимированный braille-спиннер (`⠋⠙⠹…`) во время синхронизации, `✓` в режиме ожидания, `!` при конфликте слияния. Для `✓`/`!` автоматически используются глифы Nerd Font (`nf-fa-check` / `nf-cod-warning`) если установлен `nvim-web-devicons` (подгружается при необходимости, порядок загрузки не важен); иначе — ASCII. При `repo = ''` иконка не показывается. Иконки полностью настраиваются через `config.sync_icons`.
- **Скрытие панели** — нажмите `<C-t>` (настраивается) из любого окна, чтобы скрыть проводник и дать редактору весь экран; нажмите снова, чтобы восстановить его. Удобно при написании длинных заметок.
- **Без внешних зависимостей** — чистый Lua, сторонние плагины не требуются.
- **Работает из любого каталога** — открывайте свои заметки независимо от текущего рабочего каталога.

## Требования

- Neovim ≥ 0.10
- Git (для синхронизации; необязательно, если `repo` не задан)
- Настроенный SSH-ключ для GitHub (при использовании приватного репозитория)

## Установка

### vim.pack (встроенный, Neovim 0.11+)

```lua
vim.pack.add({ src = 'https://github.com/lgick/notes.nvim' })

require('notes').setup({
  dir  = vim.fn.expand('~/.notes'),
  repo = 'git@github.com:youruser/notes.git',
})
```

### lazy.nvim

```lua
{
  'lgick/notes.nvim',
  opts = {
    dir  = vim.fn.expand('~/.notes'),
    repo = 'git@github.com:youruser/notes.git',
  },
}
```

### packer.nvim

```lua
use {
  'lgick/notes.nvim',
  config = function()
    require('notes').setup({
      dir  = vim.fn.expand('~/.notes'),
      repo = 'git@github.com:youruser/notes.git',
    })
  end,
}
```

## Конфигурация

```lua
require('notes').setup({
  -- Локальный каталог хранения заметок (он же корень рабочего дерева git).
  dir = vim.fn.expand('~/.notes'),

  -- SSH-удалёнка для синхронизации с GitHub.
  -- Оставьте пустым (''), чтобы использовать заметки локально без синхронизации git.
  repo = 'git@github.com:youruser/notes.git',

  -- Высота окна проводника в строках (без учёта статуслайна).
  list_height = 10,

  -- Сочетания клавиш (переопределяются по отдельности; не заданные сохраняют значения по умолчанию).
  keys = {
    open_file     = '<CR>',  -- проводник: папка → раскрыть/свернуть; заметка → фокус на редактор
    create        = 'a',     -- проводник: создать заметку в контекстной папке
    create_folder = 'A',     -- проводник: создать папку в контекстной папке
    delete        = 'd',     -- проводник: удалить заметку/папку под курсором
    rename        = 'r',     -- проводник: переименовать папку под курсором
    move          = 'x',     -- проводник: пометить заметку/папку под курсором для перемещения
    paste         = 'p',     -- проводник: вставить помеченный элемент в контекстную папку
    refresh       = 'R',     -- обновить дерево
    open_github   = 'O',     -- открыть репозиторий заметок в браузере
    scroll_down   = '<C-n>', -- прокрутить открытую заметку вниз
    scroll_up     = '<C-p>', -- прокрутить открытую заметку вверх
    close         = 'q',     -- закрыть заметки (работает из любого окна заметок)
    window_nav    = '<C-w>', -- префикс; затем h/j/k/l → перемещение между окнами
    toggle_panels = '<C-t>', -- скрыть/показать проводник
    change_folder = 'o',     -- проводник: папка → раскрыть/свернуть; заметка → фокус на редактор
  },

  -- Иконки статуса синхронизации в заголовке вкладки рядом с 'notes.nvim'.
  -- nil = авто: глифы Nerd Font, если загружен nvim-web-devicons, иначе Unicode.
  -- Задайте таблицу, чтобы переопределить отдельные иконки.
  sync_icons = nil,
  -- sync_icons = { idle = '✓', syncing = '⠋', conflict = '!' },

  -- Иконки на строках папок (закрыта/открыта). У заметок по умолчанию иконки нет.
  -- nil = авто: глифы Nerd Font (nf-md-folder / nf-md-folder_open), если загружен
  -- nvim-web-devicons, иначе Unicode-стрелки (▶ / ▼).
  tree_icons = nil,
  -- tree_icons = { folder = '▶', folder_open = '▼', note = '' },
})
```


## Использование

### Команды

| Команда | Описание |
|---------|----------|
| `:Notes` | Открыть вкладку заметок |

### Сочетания клавиш по умолчанию

Все клавиши настраиваются через `config.keys` (см. выше). Каждая клавиша
проводника действует на элемент под курсором, а для create/paste — на папку под
курсором или папку заметки под курсором («контекстная папка»).

| Клавиша | Действие |
|---------|----------|
| `j` / `k` | Перемещение курсора; попадание на заметку мгновенно открывает её снизу |
| `o` / `<CR>` | Папка (в т.ч. `Notes/`): раскрыть/свернуть на месте · Заметка: фокус на редактор |
| `a` | Создать заметку в контекстной папке |
| `A` | Создать папку в контекстной папке |
| `d` | Удалить заметку/папку под курсором (с подтверждением; не `Notes/`) |
| `r` | Переименовать папку под курсором (не `Notes/`) |
| `x` | Пометить заметку/папку под курсором для перемещения (не `Notes/`) |
| `p` | Вставить помеченный элемент в контекстную папку (вставка на `Notes/` — в корень) |
| `<C-n>` / `<C-p>` | Прокрутить открытую заметку вниз / вверх |
| `R` | Обновить дерево |
| `O` | Открыть репозиторий заметок в браузере |
| `<C-t>` | Скрыть / показать проводник |
| `<C-w>` затем `h`/`j`/`k`/`l` | Перемещение между окнами |
| `q` | Закрыть заметки (спросит, если в редакторе есть несохранённые изменения) |

Навигация по окнам считывает клавишу направления сразу после `<C-w>` (через `getcharstr`), поэтому не зависит от `timeoutlen`. Перемещение пространственное между проводником и редактором (`wincmd h/j/k/l`). Нажатие `<C-w>k` из редактора всегда ведёт в проводник.

### Группы подсветки

Переопределите их для настройки цветов (по умолчанию они привязаны к разумным группам):

| Группа | Привязка по умолчанию | Применяется к |
|--------|-----------------------|---------------|
| `NotesDir` | `Directory` | строки папок |
| `NotesFile` | `Normal` | строки заметок (определена для переопределения; по умолчанию построчно не применяется) |
| `NotesTitle` | `bold` (жирный) | текст заголовка в каждой строке заметки (после префикса с датой) |
| `NotesActive` | `CursorLine` | текущая открытая заметка |
| `NotesCut` | `Visual` | заметка или папка, помеченная для перемещения (`x`) |
| `NotesConflict` | undercurl, `sp` из `DiagnosticError` | волнистое подчёркивание цвета ошибки на конфликтной заметке и её папке |

`NotesActive` использует подсветку с низким приоритетом (0) на всю ширину окна (`hl_eol`), поэтому `NotesCut` (приоритет 200) и `NotesConflict` (приоритет 300) никогда ею не перекрываются, даже если обе попадают на одну строку — помеченная для перемещения или конфликтная заметка/папка остаётся видимо подсвеченной, даже будучи одновременно открытой. Окно проводника использует нативный `cursorline` (тоже на всю ширину) для строки под курсором; сам текстовый курсор терминала при этом скрыт, пока фокус в проводнике, так что именно `cursorline`, а не значок курсора, показывает текущую позицию. (Скрытие курсора зависит от того, отрисовывает ли ваш терминал/GUI цвет курсора Neovim — большинство современных терминалов это умеют, некоторые простые просто покажут свой курсор по умолчанию.)

### Плагины статуслайна

Окна проводника и редактора используют фиксированные пооконные статуслайны. Проводник показывает ` Explorer`, редактор — ` папка/заголовок [+]` когда открыта заметка (например ` Notes/Список покупок` или ` Work/Идея проекта`), и ` Editor` когда ни одна заметка не выбрана. Если вы используете плагин статуслайна (lualine и т.п.), который переопределяет пооконные статуслайны, добавьте тип файла `NotesExplorer` в его список исключений, а окно редактора исключите по типу файла (`markdown`) или по проверке пути буфера.

Вкладка заметок имеет заголовок `notes.nvim` плюс индикатор статуса (например `notes.nvim ✓` или вращающийся `notes.nvim ⠋` во время синхронизации). Заголовок закреплён в переменной вкладки `t:title`, поэтому плагины tabline, которые её читают, покажут правильное имя независимо от того, какое внутреннее окно в фокусе. Только если у вас **не** задан `tabline`, плагин устанавливает свой собственный (восстанавливается при закрытии). Для настройки иконок см. `config.sync_icons`.

### Структура файлов

```
~/.notes/                  ← config.dir
  20260626223010.md        ← заметка: непрозрачный ID-файл (.md); заголовок = первая строка
  Work/                    ← папка (поддерживается любая глубина вложенности)
    20260625101500.md      ← заметка внутри папки
    Projects/               ← подпапка, раскрывается через `o`
      20260701090000.md
    .gitkeep               ← скрытый маркер, чтобы пустая папка всё равно коммитилась
  Personal/
    .gitkeep
```

Каждая заметка — это `.md`-файл с ID-именем; её заголовок в дереве читается из первой непустой строки содержимого. Заметки без папки лежат прямо под корневой строкой `Notes/`. Папки могут вкладываться на любую глубину и раскрываются/сворачиваются на месте по `o`, так же как и сама `Notes/`. Создавайте заметку через `a`, папку через `A` (обе — внутри контекстной папки); переименовывайте папку через `r`, удаляйте через `d`; перемещайте заметку или целую папку (со всем содержимым) между папками через `x` (пометить), затем `p` (вставить в контекстную папку, или на `Notes/` — в корень) — папку нельзя переместить в саму себя, в одну из своих подпапок, а корень переместить нельзя вовсе.

### Поведение синхронизации Git

| Событие | Действие |
|---------|----------|
| Каждое открытие | Восстановить отслеживаемые файлы, удалённые вне плагина (`git checkout -- <deleted>`) |
| Первый `:Notes` за сессию | `git clone`, если отсутствует, затем `git pull` (merge) |
| Последующие `:Notes` | Только восстановление; без сетевых вызовов (уже синхронизировано) |
| Сохранение файла (`:w`) | Интерфейс обновляется мгновенно; затем `git commit` → `git pull` (merge) → `git push` |
| Создание / удаление / перемещение / переименование | Немедленный `git commit` → `git pull` (merge) → `git push` |
| Закрытие заметок (`q`) | По желанию сохраняет открытый буфер, затем `git commit` → `git pull` (merge) → `git push` |

### Конфликты

**Диалогов нет.** Когда `git pull` не может слить изменения автоматически, конфликт остаётся в заметке обычными git-маркерами, а репозиторий переходит в штатное состояние слияния:

```
<<<<<<< HEAD
ваша локальная строка
=======
версия с GitHub
>>>>>>> origin/main
```

Заголовок конфликтной заметки — и строка папки, в которой она лежит (любого предка, рекурсивно) — подчёркиваются волнистой линией цвета ошибки (`NotesConflict`), так что сразу видно, какие заметки требуют внимания, даже пока их папка свёрнута. Чтобы разрешить: откройте заметку, уберите маркеры и сохраните (`:w`). Это завершит слияние и запушит. Заметка с оставшимися маркерами никогда не коммитится.

Пока заметка в конфликте, её (и её папки) **перемещение / переименование / удаление** блокируется сообщением `Resolve the conflict first` — перемещение файла во время слияния повредило бы индекс git. Конфликт «изменено/удалено» (одна сторона правила, другая удалила) разрешается автоматически в пользу сохранения файла, поэтому синхронизация никогда не застревает.

Несколько быстрых CRUD-действий сериализуются: в каждый момент выполняется не более одной цепочки git, плюс одна отложенная цепочка, которая захватывает всё, что накопилось, пока первая была в работе.

Задайте `repo = ''`, чтобы отключить все операции git.
