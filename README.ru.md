# notes.nvim

*[English version](README.md)*

Лёгкий плагин для Neovim, который ведёт заметки в отдельной вкладке — по образу
приложения **macOS Notes** — с опциональной синхронизацией с GitHub через Git (SSH).

```
──────────────┬──────────────────────────────────
 Folders      │ Notes                            ← статуслайны
 Notes         │ 26.06.2026 - Список покупок
 Work          │ 25.06.2026 - Идея проекта
 Personal      │ 24.06.2026 - Отчёт
──────────────┴──────────────────────────────────
 Editor                                           ← статуслайн
 # Список покупок

 - [ ] Молоко
 - [ ] Позвонить в банк
```

## Возможности

- **Двухпанельный интерфейс в стиле macOS Notes** — открывается в новой полноэкранной вкладке: **папки** (левая колонка) и **заметки** (правая колонка) сверху, редактор снизу. Закрытие заметок закрывает вкладку.
- **Заголовок из содержимого** — у заметки нет задаваемого вручную имени файла. Её заголовок — это **первая непустая строка** текста; пустая заметка называется **«New Note»** и всегда закреплена вверху списка. На диске каждая заметка — это непрозрачный ID-файл (расширение `.md`, например `20260627143000.md`), поэтому изменение заголовка не засоряет историю git и не вызывает коллизий. В колонке заметок показывается `дд.мм.гггг - Заголовок`, отсортировано по времени изменения (сначала новые). **Заголовок в списке обновляется на лету по мере набора текста**, без сохранения.
- **Папки одного уровня вложенности** — в колонке папок перечислены **«Notes»** (корень: заметки без папки) плюс ваши папки, отсортированные так, что папка с самой недавно изменённой заметкой идёт первой. Выбор папки фильтрует колонку заметок по ней. Пустые папки поддерживаются через скрытый `.gitkeep`, чтобы они коммитились и синхронизировались.
- **Перемещение курсором** — нажмите `x` на заметке, чтобы пометить её (подсвечивается цветом выделения), затем перейдите к папке в колонке папок и нажмите `p`, чтобы переместить заметку туда.
- **Нативное редактирование** — окно редактора ведёт себя как обычное окно `markdown`-файла (`number`, `cursorline`, `signcolumn`), поэтому глобальные стили на `InsertEnter`/`InsertLeave` и плагины статуслайна работают внутри него.
- **Мгновенное обновление интерфейса** — список заметок обновляется сразу при `:w` (порядок сортировки, заголовок); синхронизация git идёт в фоне.
- **Полное управление** — одинаковые клавиши в каждой колонке: `a` создаёт (заметку в колонке заметок, папку в колонке папок), `d` удаляет; заметки также поддерживают перемещение (`x` + `p`), папки — переименование (`r`); обновление (`R`). **Каждое создание/удаление/перемещение/переименование сразу коммитится и пушится в GitHub.**
- **Настраиваемые сочетания клавиш** — каждое действие, клавиша закрытия и клавиши переключения панелей переопределяются через `config.keys`.
- **Синхронизация Git** — при первом открытии: `git clone` (если каталога нет), затем `git pull --rebase --autostash`. При `:w`: commit+push. При любом создании/удалении/перемещении/переименовании: немедленный commit+push. При закрытии (`q`): commit+push оставшихся изменений.
- **Запрос о несохранённых изменениях** — нажатие `q` при несохранённых изменениях в редакторе показывает диалог **Save / Discard / Cancel** вместо тихого сохранения или сброса. Выбор **Discard** перезагружает сохранённую версию с диска.
- **Защита от сбоев** — при каждом открытии отслеживаемые файлы, удалённые вне плагина (например, случайным `rm`), восстанавливаются из последнего коммита перед любым push, так что пустое рабочее дерево никогда не попадёт в удалённый репозиторий.
- **Без внешних зависимостей** — чистый Lua, сторонние плагины не требуются.
- **Работает из любого каталога** — открывайте свои заметки независимо от текущего рабочего каталога.

## Требования

- Neovim ≥ 0.10
- Git (для синхронизации; необязательно, если `repo` не задан)
- Настроенный SSH-ключ для GitHub (при использовании приватного репозитория)

## Установка

### vim.pack (встроенный, Neovim 0.11+)

```lua
vim.pack.add({ src = 'https://github.com/lgick/notes.nvim' })

require('notes').setup({
  dir  = vim.fn.expand('~/.notes'),
  repo = 'git@github.com:youruser/notes.git',
})
```

### lazy.nvim

```lua
{
  'lgick/notes.nvim',
  opts = {
    dir  = vim.fn.expand('~/.notes'),
    repo = 'git@github.com:youruser/notes.git',
  },
}
```

### packer.nvim

```lua
use {
  'lgick/notes.nvim',
  config = function()
    require('notes').setup({
      dir  = vim.fn.expand('~/.notes'),
      repo = 'git@github.com:youruser/notes.git',
    })
  end,
}
```

## Конфигурация

```lua
require('notes').setup({
  -- Локальный каталог хранения заметок (он же корень рабочего дерева git).
  dir = vim.fn.expand('~/.notes'),

  -- SSH-удалёнка для синхронизации с GitHub.
  -- Оставьте пустым (''), чтобы использовать заметки локально без синхронизации git.
  repo = 'git@github.com:youruser/notes.git',

  -- Высота строки папок/заметок в строках (без учёта статуслайна).
  list_height = 10,

  -- Ширина колонки папок.
  folders_width = 25,

  -- Сочетания клавиш (переопределяются по отдельности; не заданные сохраняют значения по умолчанию).
  keys = {
    open_file   = '<CR>',  -- папки: фокус на колонку заметок; заметки: фокус на редактор
    create      = 'a',     -- папки: создать папку; заметки: создать заметку
    delete      = 'd',     -- папки: удалить папку; заметки: удалить заметку (с подтверждением)
    rename      = 'r',     -- папки: переименовать выбранную папку
    move        = 'x',     -- заметки: пометить заметку для перемещения
    paste       = 'p',     -- папки: переместить помеченную заметку в выбранную папку
    refresh     = 'R',     -- обновить список
    open_github = 'O',     -- открыть репозиторий заметок в браузере
    scroll_down = '<C-n>', -- заметки: прокрутить открытую заметку вниз
    scroll_up   = '<C-p>', -- заметки: прокрутить открытую заметку вверх
    close       = 'q',     -- закрыть заметки (работает из любого окна заметок)
    window_nav  = '<C-w>', -- префикс; затем h/j/k/l → перемещение между окнами
  },
})
```

### Сочетание клавиш (рекомендуемое)

```lua
vim.keymap.set('n', '<leader>m', '<cmd>Notes<CR>', { desc = 'Notes' })
```

## Использование

### Команды

| Команда | Описание |
|---------|----------|
| `:Notes` | Открыть вкладку заметок |

### Сочетания клавиш по умолчанию

Все клавиши настраиваются через `config.keys` (см. выше).

| Клавиша | Действие | Где |
|---------|----------|-----|
| `j` / `k` | Перемещение курсора → фильтр заметок по этой папке | папки |
| `a` | Создать папку | папки |
| `r` | Переименовать выбранную папку | папки |
| `d` | Удалить выбранную папку (с подтверждением) | папки |
| `<CR>` | Фокус на колонку заметок | папки |
| `p` | Переместить помеченную заметку в эту папку | папки |
| `j` / `k` | Перемещение курсора + мгновенное открытие заметки | заметки |
| `<CR>` | Фокус на окно редактора | заметки |
| `a` | Создать новую заметку в текущей папке (или в корне) | заметки |
| `d` | Удалить выбранную заметку (с подтверждением) | заметки |
| `x` | Пометить заметку для перемещения (затем перейти к папке и нажать `p`) | заметки |
| `<C-n>` / `<C-p>` | Прокрутить открытую заметку вниз / вверх | заметки |
| `R` | Обновить список | папки / заметки |
| `O` | Открыть репозиторий заметок в браузере | папки / заметки |
| `<C-w>` затем `h`/`j`/`k`/`l` | Перемещение между окнами | любое |
| `q` | Закрыть заметки (спросит, если в редакторе есть несохранённые изменения) | любое |

Навигация по окнам считывает клавишу направления сразу после `<C-w>` (через `getcharstr`), поэтому не зависит от `timeoutlen`. Перемещение пространственное между тремя окнами (`wincmd h/j/k/l`). Нажатие `<C-w>k` из редактора всегда ведёт в колонку заметок (а не папок).

### Группы подсветки

Переопределите их для настройки цветов (по умолчанию они привязаны к разумным группам):

| Группа | Привязка по умолчанию | Применяется к |
|--------|-----------------------|---------------|
| `NotesDir` | `Directory` | строки папок |
| `NotesFile` | `Normal` | строки заметок (определена для переопределения; по умолчанию построчно не применяется) |
| `NotesActive` | `CursorLine` | текущая открытая заметка в колонке заметок |
| `NotesDirActive` | вычисляется | выбранная папка в колонке папок |
| `NotesCut` | `Visual` | заметка, помеченная для перемещения (`x`) |

`NotesDirActive` вычисляется при открытии из разрешённых цветов `Directory` (fg) и `CursorLine` (bg), объединяя цвет папки с фоном выделения. Переопределите её явным вызовом `nvim_set_hl` в своей конфигурации после setup.

`NotesActive` и `NotesDirActive` используют `line_hl_group`, чтобы заполнять всю ширину строки и оставаться видимыми, когда фокус в другом месте. У `NotesCut` приоритет (200) выше, чем у `NotesActive` (0), поэтому она никогда не перекрывается ею, даже если обе попадают в одну строку. Окна папок и заметок не используют `cursorline`; текущая позиция обозначается курсором терминала.

### Плагины статуслайна

Все три окна используют фиксированные пооконные статуслайны (` Folders`, ` Notes`, ` Editor`). Если вы используете плагин статуслайна (lualine и т.п.), который переопределяет пооконные статуслайны, добавьте типы файлов `NotesFolders` и `NotesList` в его список исключений, а окно редактора исключите по типу файла (`markdown`) или по проверке пути буфера.

### Структура файлов

```
~/.notes/                  ← config.dir
  20260626223010.md        ← заметка: непрозрачный ID-файл (.md); заголовок = первая строка
  Work/                    ← папка (один уровень вложенности)
    20260625101500.md      ← заметка внутри папки
    .gitkeep               ← скрытый маркер, чтобы пустая папка всё равно коммитилась
  Personal/
    .gitkeep
```

Каждая заметка — это `.md`-файл с ID-именем; её заголовок в списке читается из первой непустой строки содержимого. Виртуальная запись «Notes» в колонке папок — это корень репозитория (заметки без папки). Папки одного уровня вложенности — создавайте через `a`, переименовывайте через `r`, удаляйте через `d`; перемещайте заметки между папками через `x` (пометить), затем `p` (вставить в выбранную папку).

### Поведение синхронизации Git

| Событие | Действие |
|---------|----------|
| Каждое открытие | Восстановить отслеживаемые файлы, удалённые вне плагина (`git checkout -- <deleted>`) |
| Первый `:Notes` за сессию | `git clone`, если отсутствует, затем `git pull --rebase --autostash` |
| Последующие `:Notes` | Только восстановление; без сетевых вызовов (уже синхронизировано) |
| Сохранение файла (`:w`) | Интерфейс обновляется мгновенно; затем fetch → reconcile → `git add -A` → `git commit` → `git push` |
| Создание / удаление / перемещение / переименование | Немедленный fetch → reconcile → `git add -A` → `git commit` → `git push` |
| Закрытие заметок (`q`) | По желанию сохраняет открытый буфер, затем fetch → reconcile → `git add -A` → `git commit` → `git push` |

**Reconcile** означает: если удалёнка впереди, `git stash push` → `git pull --ff-only` → `git stash pop`. Если pop вызывает конфликт, появляется диалог:

```
[notes.nvim] GitHub updated: work/todo.md
Local changes will overwrite. Push?
[Yes] [No]
```

- **Yes** — ваша локальная версия сохраняется и пушится в GitHub. Это обрабатывает все типы конфликтов: `UU` (изменены обе версии), `DU` (GitHub удалил, вы изменили), `UD` (GitHub изменил, вы удалили) и т.д. Список файлов обновляется автоматически.
- **No** — ваши локальные изменения отбрасываются; редактор перезагружает версию GitHub с диска, список файлов обновляется автоматически.

Несколько быстрых CRUD-действий сериализуются: в каждый момент выполняется не более одной цепочки git, плюс одна отложенная цепочка, которая захватывает всё, что накопилось, пока первая была в работе.

Задайте `repo = ''`, чтобы отключить все операции git.
