# notes.nvim

*[English version](README.md)*

Лёгкий плагин для Neovim, который ведёт заметки в отдельной вкладке — по образу
приложения **macOS Notes** — с опциональной синхронизацией с GitHub через Git (SSH).

<video src="https://github.com/user-attachments/assets/7ff882ff-7cfc-4739-8765-e9e4ddc6fd5f" autoplay loop muted playsinline></video>

```
──────────────┬──────────────────────────────────
 Folders      │ Notes                            ← статуслайны
 Notes/        │ 26.06.2026 - Список покупок
 ├─ Work/      │ 25.06.2026 - Идея проекта
 └─ Personal/  │ 24.06.2026 - Отчёт
──────────────┴──────────────────────────────────
 Notes/Список покупок                             ← статуслайн (папка/заголовок)
 # Список покупок

 - [ ] Молоко
 - [ ] Позвонить в банк
```

Папки вкладываются на любую глубину и показываются по одному уровню за раз (`o` — войти / подняться):

```
────────────────┬────────────────────────────────
 Folders        │ Notes
 Notes/Work/ .. │ 26.06.2026 - Заметки спринта
 ├─ Projects/   │
 └─ Archive/    │
────────────────┴────────────────────────────────
```

## Возможности

- **Двухпанельный интерфейс в стиле macOS Notes** — открывается в новой полноэкранной вкладке: **папки** (левая колонка) и **заметки** (правая колонка) сверху, редактор снизу. Закрытие заметок закрывает вкладку.
- **Заголовок из содержимого** — у заметки нет задаваемого вручную имени файла. Её заголовок — это **первая непустая строка** текста; пустая заметка называется **«New Note»** и всегда закреплена вверху списка. На диске каждая заметка — это непрозрачный ID-файл (расширение `.md`, например `20260627143000.md`), поэтому изменение заголовка не засоряет историю git и не вызывает коллизий. В колонке заметок показывается `дд.мм.гггг - Заголовок`, отсортировано по времени изменения (сначала новые). **Заголовок в списке обновляется на лету по мере набора текста**, без сохранения.
- **Вложенные папки, навигация drill-down** — колонка папок показывает по одному уровню за раз: первая строка — текущий уровень (**«Notes»** в корне, или `Notes/<путь>/ ..` после входа в папку), ниже — её непосредственные подпапки, отсортированные так, что папка, в чьём поддереве самая недавно изменённая заметка, идёт первой. Нажмите `o` на подпапке, чтобы войти в неё и загрузить её заметки в колонку заметок, или `o` на первой строке, чтобы подняться на уровень выше. Одно лишь перемещение курсора не перезагружает колонку заметок — это делает только вход в папку через `o`; для колонки папок используется нативная подсветка `cursorline`, показывающая строку под курсором. Новые папки (`a`) создаются внутри текущего уровня. Пустые папки поддерживаются через скрытый `.gitkeep`, чтобы они коммитились и синхронизировались.
- **Перемещение курсором** — нажмите `x` на заметке (или на папке), чтобы пометить её (подсвечивается цветом выделения); повторное нажатие `x` на том же элементе отменяет перемещение. Для заметки перейдите к папке в колонке папок и нажмите `p`, чтобы переместить её туда. Для папки перемещайтесь внутри самой колонки папок (входите/выходите через `o`, двигайте курсор) к папке-приёмнику и нажмите `p` там же. Папка-приёмник становится выбранной и поднимается наверх колонки папок; при перемещении папки колонка также заходит в папку-приёмник, чтобы перемещённая папка отобразилась как её дочерний элемент. Пометить можно только подпапку (дочернюю строку) — папку, которую вы сейчас просматриваете (первая строка, включая корень «Notes»), пометить изнутри неё самой нельзя. Также папку нельзя переместить в саму себя или в одну из своих подпапок.
- **Нативное редактирование** — окно редактора ведёт себя как обычное окно `markdown`-файла (`number`, `cursorline`, `signcolumn`), поэтому глобальные стили на `InsertEnter`/`InsertLeave` и плагины статуслайна работают внутри него.
- **Мгновенное обновление интерфейса** — список заметок обновляется сразу при `:w` (порядок сортировки, заголовок); синхронизация git идёт в фоне.
- **Полное управление** — одинаковые клавиши в каждой колонке: `a` создаёт (заметку в колонке заметок, папку в колонке папок), `d` удаляет, `x` помечает для перемещения, `p` вставляет; папки также поддерживают переименование (`r`); обновление (`R`). **Каждое создание/удаление/перемещение/переименование сразу коммитится и пушится в GitHub.**
- **Настраиваемые сочетания клавиш** — каждое действие, клавиша закрытия и клавиши переключения панелей переопределяются через `config.keys`.
- **Синхронизация Git** — при первом открытии: `git clone` (если каталога нет), затем `git pull` (merge). При `:w`: commit + merge + push. При любом создании/удалении/перемещении/переименовании: немедленный commit + merge + push. При закрытии (`q`): commit + merge + push оставшихся изменений.
- **Конфликты остаются в файле** — конфликт слияния оставляется обычными git-маркерами прямо в заметке (без диалога). Заголовок конфликтной заметки и имя её папки подчёркиваются волнистой линией цвета ошибки; уберите маркеры, сохраните — слияние завершится и запушится. Перемещение/переименование/удаление конфликтной заметки блокируется до разрешения.
- **Запрос о несохранённых изменениях** — нажатие `q` при несохранённых изменениях в редакторе показывает диалог **Save / Discard / Cancel** вместо тихого сохранения или сброса. Выбор **Discard** перезагружает сохранённую версию с диска.
- **Защита от сбоев** — при каждом открытии отслеживаемые файлы, удалённые вне плагина (например, случайным `rm`), восстанавливаются из последнего коммита перед любым push, так что пустое рабочее дерево никогда не попадёт в удалённый репозиторий.
- **Иконка статуса синхронизации** — в заголовке вкладки Neovim рядом с `notes.nvim` отображается индикатор: анимированный braille-спиннер (`⠋⠙⠹…`) во время синхронизации, `✓` в режиме ожидания, `!` при конфликте слияния. Для `✓`/`!` автоматически используются глифы Nerd Font (`nf-fa-check` / `nf-cod-warning`) если установлен `nvim-web-devicons` (подгружается при необходимости, порядок загрузки не важен); иначе — ASCII. При `repo = ''` иконка не показывается. Иконки полностью настраиваются через `config.sync_icons`.
- **Скрытие панелей** — нажмите `<C-t>` (настраивается) из любого окна, чтобы скрыть колонки Folders и Notes и дать редактору весь экран; нажмите снова, чтобы восстановить их. Удобно при написании длинных заметок.
- **Без внешних зависимостей** — чистый Lua, сторонние плагины не требуются.
- **Работает из любого каталога** — открывайте свои заметки независимо от текущего рабочего каталога.

## Требования

- Neovim ≥ 0.10
- Git (для синхронизации; необязательно, если `repo` не задан)
- Настроенный SSH-ключ для GitHub (при использовании приватного репозитория)

## Установка

### vim.pack (встроенный, Neovim 0.11+)

```lua
vim.pack.add({ src = 'https://github.com/lgick/notes.nvim' })

require('notes').setup({
  dir  = vim.fn.expand('~/.notes'),
  repo = 'git@github.com:youruser/notes.git',
})
```

### lazy.nvim

```lua
{
  'lgick/notes.nvim',
  opts = {
    dir  = vim.fn.expand('~/.notes'),
    repo = 'git@github.com:youruser/notes.git',
  },
}
```

### packer.nvim

```lua
use {
  'lgick/notes.nvim',
  config = function()
    require('notes').setup({
      dir  = vim.fn.expand('~/.notes'),
      repo = 'git@github.com:youruser/notes.git',
    })
  end,
}
```

## Конфигурация

```lua
require('notes').setup({
  -- Локальный каталог хранения заметок (он же корень рабочего дерева git).
  dir = vim.fn.expand('~/.notes'),

  -- SSH-удалёнка для синхронизации с GitHub.
  -- Оставьте пустым (''), чтобы использовать заметки локально без синхронизации git.
  repo = 'git@github.com:youruser/notes.git',

  -- Высота строки папок/заметок в строках (без учёта статуслайна).
  list_height = 10,

  -- Ширина колонки папок.
  folders_width = 25,

  -- Сочетания клавиш (переопределяются по отдельности; не заданные сохраняют значения по умолчанию).
  keys = {
    open_file   = '<CR>',  -- папки: фокус на колонку заметок; заметки: фокус на редактор
    create      = 'a',     -- папки: создать папку; заметки: создать заметку
    delete      = 'd',     -- папки: удалить папку; заметки: удалить заметку (с подтверждением)
    rename      = 'r',     -- папки: переименовать выбранную папку
    move        = 'x',     -- заметки: пометить заметку для перемещения
    paste       = 'p',     -- папки: переместить помеченную заметку в выбранную папку
    refresh     = 'R',     -- обновить список
    open_github = 'O',     -- открыть репозиторий заметок в браузере
    scroll_down = '<C-n>', -- заметки: прокрутить открытую заметку вниз
    scroll_up   = '<C-p>', -- заметки: прокрутить открытую заметку вверх
    close         = 'q',     -- закрыть заметки (работает из любого окна заметок)
    window_nav    = '<C-w>', -- префикс; затем h/j/k/l → перемещение между окнами
    toggle_panels = '<C-t>', -- скрыть/показать колонки Folders + Notes
    change_folder = 'o',     -- папки: войти в папку под курсором / подняться с первой строки
  },

  -- Иконки статуса синхронизации в заголовке вкладки рядом с 'notes.nvim'.
  -- nil = авто: глифы Nerd Font, если загружен nvim-web-devicons, иначе Unicode.
  -- Задайте таблицу, чтобы переопределить отдельные иконки.
  sync_icons = nil,
  -- sync_icons = { idle = '✓', syncing = '⠋', conflict = '!' },
})
```

### Сочетание клавиш (рекомендуемое)

```lua
vim.keymap.set('n', '<leader>m', '<cmd>Notes<CR>', { desc = 'Notes' })
```

## Использование

### Команды

| Команда | Описание |
|---------|----------|
| `:Notes` | Открыть вкладку заметок |

### Сочетания клавиш по умолчанию

Все клавиши настраиваются через `config.keys` (см. выше).

| Клавиша | Действие | Где |
|---------|----------|-----|
| `j` / `k` | Перемещение курсора (навигация; `o` — войти в папку и загрузить её) | папки |
| `a` | Создать папку внутри текущего уровня | папки |
| `r` | Переименовать выбранную папку | папки |
| `d` | Удалить выбранную папку (с подтверждением) | папки |
| `o` | Войти в папку под курсором / подняться с первой строки | папки |
| `<CR>` | Фокус на колонку заметок | папки |
| `x` | Пометить папку для перемещения (затем перейти внутри Folders и нажать `p`) | папки |
| `p` | Переместить помеченную заметку/папку в эту папку | папки |
| `j` / `k` | Перемещение курсора + мгновенное открытие заметки | заметки |
| `<CR>` | Фокус на окно редактора | заметки |
| `a` | Создать новую заметку в текущей папке (или в корне) | заметки |
| `d` | Удалить выбранную заметку (с подтверждением) | заметки |
| `x` | Пометить заметку для перемещения (затем перейти к папке и нажать `p`) | заметки |
| `<C-n>` / `<C-p>` | Прокрутить открытую заметку вниз / вверх | заметки |
| `R` | Обновить список | папки / заметки |
| `O` | Открыть репозиторий заметок в браузере | папки / заметки |
| `<C-t>` | Скрыть / показать колонки Folders + Notes | любое |
| `<C-w>` затем `h`/`j`/`k`/`l` | Перемещение между окнами | любое |
| `q` | Закрыть заметки (спросит, если в редакторе есть несохранённые изменения) | любое |

Навигация по окнам считывает клавишу направления сразу после `<C-w>` (через `getcharstr`), поэтому не зависит от `timeoutlen`. Перемещение пространственное между тремя окнами (`wincmd h/j/k/l`). Нажатие `<C-w>k` из редактора всегда ведёт в колонку заметок (а не папок).

### Группы подсветки

Переопределите их для настройки цветов (по умолчанию они привязаны к разумным группам):

| Группа | Привязка по умолчанию | Применяется к |
|--------|-----------------------|---------------|
| `NotesDir` | `Directory` | строки папок |
| `NotesFile` | `Normal` | строки заметок (определена для переопределения; по умолчанию построчно не применяется) |
| `NotesTitle` | `bold` (жирный) | текст заголовка в каждой строке заметки (после префикса с датой) |
| `NotesActive` | `CursorLine` | текущая открытая заметка в колонке заметок |
| `NotesCut` | `Visual` | заметка или папка, помеченная для перемещения (`x`) |
| `NotesConflict` | undercurl, `sp` из `DiagnosticError` | волнистое подчёркивание цвета ошибки на конфликтной заметке и её папке |

`NotesActive` и `NotesDir` используют подсветку с низким приоритетом (0) поверх текста строки, поэтому `NotesCut` (приоритет 200) и `NotesConflict` (приоритет 300) никогда ею не перекрываются, даже если обе попадают на одну строку — помеченная для перемещения или конфликтная заметка/папка остаётся видимо подсвеченной, даже будучи под курсором. Окно заметок не использует `cursorline`; текущая позиция обозначается курсором терминала. Колонка папок — исключение: она использует нативную подсветку `cursorline` как индикатор навигации, поскольку перемещение курсора там больше не меняет содержимое колонки заметок.

### Плагины статуслайна

Все три окна используют фиксированные пооконные статуслайны. Колонка папок показывает ` Folders`, колонка заметок — ` Notes`, редактор — ` папка/заголовок [+]` когда открыта заметка (например ` Notes/Список покупок` или ` Work/Идея проекта`), и ` Editor` когда ни одна заметка не выбрана. Если вы используете плагин статуслайна (lualine и т.п.), который переопределяет пооконные статуслайны, добавьте типы файлов `NotesFolders` и `NotesList` в его список исключений, а окно редактора исключите по типу файла (`markdown`) или по проверке пути буфера.

Вкладка заметок имеет заголовок `notes.nvim` плюс индикатор статуса (например `notes.nvim ✓` или вращающийся `notes.nvim ⠋` во время синхронизации). Заголовок закреплён в переменной вкладки `t:title`, поэтому плагины tabline, которые её читают, покажут правильное имя независимо от того, какое внутреннее окно в фокусе. Только если у вас **не** задан `tabline`, плагин устанавливает свой собственный (восстанавливается при закрытии). Для настройки иконок см. `config.sync_icons`.

### Структура файлов

```
~/.notes/                  ← config.dir
  20260626223010.md        ← заметка: непрозрачный ID-файл (.md); заголовок = первая строка
  Work/                    ← папка (поддерживается любая глубина вложенности)
    20260625101500.md      ← заметка внутри папки
    Projects/               ← подпапка, вход через `o`
      20260701090000.md
    .gitkeep               ← скрытый маркер, чтобы пустая папка всё равно коммитилась
  Personal/
    .gitkeep
```

Каждая заметка — это `.md`-файл с ID-именем; её заголовок в списке читается из первой непустой строки содержимого. Виртуальная запись «Notes» в колонке папок — это корень репозитория (заметки без папки). Папки могут вкладываться на любую глубину; колонка папок показывает один уровень за раз — нажмите `o`, чтобы войти или подняться. Создавайте через `a` (внутри текущего уровня), переименовывайте через `r`, удаляйте через `d`; перемещайте заметки между папками через `x` (пометить), затем `p` (вставить в выбранную папку). Целые папки переносятся так же (`x`, затем `p`), только переход к папке-приёмнику происходит внутри самой колонки папок, а не переключением колонки; пометить можно только подпапку, а не просматриваемую папку, и перенести папку в саму себя или в одну из своих подпапок нельзя.

### Поведение синхронизации Git

| Событие | Действие |
|---------|----------|
| Каждое открытие | Восстановить отслеживаемые файлы, удалённые вне плагина (`git checkout -- <deleted>`) |
| Первый `:Notes` за сессию | `git clone`, если отсутствует, затем `git pull` (merge) |
| Последующие `:Notes` | Только восстановление; без сетевых вызовов (уже синхронизировано) |
| Сохранение файла (`:w`) | Интерфейс обновляется мгновенно; затем `git commit` → `git pull` (merge) → `git push` |
| Создание / удаление / перемещение / переименование | Немедленный `git commit` → `git pull` (merge) → `git push` |
| Закрытие заметок (`q`) | По желанию сохраняет открытый буфер, затем `git commit` → `git pull` (merge) → `git push` |

### Конфликты

**Диалогов нет.** Когда `git pull` не может слить изменения автоматически, конфликт остаётся в заметке обычными git-маркерами, а репозиторий переходит в штатное состояние слияния:

```
<<<<<<< HEAD
ваша локальная строка
=======
версия с GitHub
>>>>>>> origin/main
```

Заголовок конфликтной заметки — и имя папки, в которой она лежит — подчёркиваются волнистой линией цвета ошибки (`NotesConflict`) в обеих колонках, так что сразу видно, какие заметки требуют внимания. Чтобы разрешить: откройте заметку, уберите маркеры и сохраните (`:w`). Это завершит слияние и запушит. Заметка с оставшимися маркерами никогда не коммитится.

Пока заметка в конфликте, её (и её папки) **перемещение / переименование / удаление** блокируется сообщением `Resolve the conflict first` — перемещение файла во время слияния повредило бы индекс git. Конфликт «изменено/удалено» (одна сторона правила, другая удалила) разрешается автоматически в пользу сохранения файла, поэтому синхронизация никогда не застревает.

Несколько быстрых CRUD-действий сериализуются: в каждый момент выполняется не более одной цепочки git, плюс одна отложенная цепочка, которая захватывает всё, что накопилось, пока первая была в работе.

Задайте `repo = ''`, чтобы отключить все операции git.
